## 1. 产品概述
小型电脑维修店管理系统，用于管理维修进度、零件库存、客户信息和月度统计，解决手工记录混乱、库存不准、催取遗漏等问题。
- 目标用户：维修店老板及员工
- 核心价值：提升运营效率、减少错漏、数据可视化决策

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 店员 | 默认登录 | 全部功能：维修单管理、库存管理、统计查看 |

### 2.2 功能模块
1. **仪表盘**：待处理提醒、超期维修单、今日统计、快速入口
2. **维修单管理**：新建维修单、状态流转、维修记录、费用结算
3. **零件库存管理**：零件列表、入库出库、库存预警
4. **统计报表**：月度维修量、故障类型统计、零件消耗排行

## 3. 核心流程
客户送修设备 → 店员创建维修单填写基础信息 → 设备检查后填写维修方案和报价 → 客户确认同意 → 开始维修，更换零件并从库存扣减 → 维修完成标记待取件 → 超过3天未取自动标红提醒 → 客户取机结算费用（零件费+工时费）→ 维修单完成。

```mermaid
flowchart TD
    A["客户送修"] --> B["创建维修单<br/>填写客户/设备/故障"]
    B --> C["填写维修方案和报价"]
    C --> D{"客户同意?"}
    D -->|否| E["维修单取消"]
    D -->|是| F["开始维修"]
    F --> G["更换零件<br/>库存扣减"]
    G --> H["维修完成<br/>标记待取件"]
    H --> I{"超过3天未取?"}
    I -->|是| J["标红提醒<br/>一键拨号催取"]
    I -->|否| K["客户取机"]
    J --> K
    K --> L["结算费用<br/>零件费+工时费"]
    L --> M["维修单完成"]
```

## 4. 用户界面设计

### 4.1 设计风格
- 主色调：深青色 #0D9488（专业、稳重），辅助色：琥珀色 #F59E0B（提醒），危险色：红色 #EF4444（超期警告）
- 按钮风格：圆角矩形，悬浮阴影效果
- 字体：标题使用 "Noto Sans SC" 粗体，正文使用系统无衬线字体
- 布局风格：左侧导航栏 + 右侧内容区，卡片式布局
- 图标风格：使用 Lucide 线性图标

### 4.2 页面设计概览
| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 仪表盘 | 超期提醒 | 红色渐变背景卡片，闪烁效果，电话按钮 |
| 仪表盘 | 统计卡片 | 带图标数据卡片，数字动画 |
| 维修单列表 | 状态标签 | 彩色胶囊标签（灰/蓝/绿/黄/红） |
| 维修单详情 | 时间线 | 垂直时间线展示状态流转 |
| 零件库存 | 库存预警 | 黄色背景行，预警图标 |
| 统计报表 | 图表 | 渐变柱状图、环形饼图 |

### 4.3 响应式
桌面端优先设计（最小宽度1280px），平板自适应，主要在电脑端使用。
